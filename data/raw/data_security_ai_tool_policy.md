# Data Security and AI Tool Usage Policy

## Purpose and Scope

This policy governs employee use of AI tools when handling company information. It applies to employees, contractors, and temporary workers who access company data.

Information Security owns this policy. The Data Protection Office must review exceptions involving personal data or cross-border data transfer.

## Approved AI Tools

Employees may use approved enterprise AI tools listed in the internal software catalog for business tasks that do not expose prohibited data.

Public AI tools are not approved for company confidential information, customer personal data, employee personal data, source code secrets, or regulated business records.

## Prohibited Data

Employees must not enter customer personal data, employee personal data, confidential company information, security credentials, source code secrets, or unreleased financial information into public AI tools.

Screenshots of employee IDs, customer records, or internal security tickets are treated as personal or confidential data and must not be uploaded to public AI tools.

## Exception Process

AI tool exceptions require review by Information Security. If the exception involves personal data, the Data Protection Office must also approve the request before use.

Exception requests must describe the tool, data category, business purpose, retention behavior, and risk controls.

## Incident Reporting

If prohibited data is accidentally entered into an AI tool, the employee must stop using the tool, preserve the prompt and output if safe to do so, and report the incident to Information Security immediately.

Information Security determines whether Legal, Compliance, or the Data Protection Office must be notified.
