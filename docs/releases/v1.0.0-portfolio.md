## Release Positioning

This release packages Enterprise KnowledgeOps Copilot as a final portfolio-ready local demo. The project is not a generic chatbot. It is a local-first enterprise knowledge operations platform that turns synthetic corporate documents into searchable, citation-backed, graph-inspectable, query-routable, answerable, evaluable, and feedback-governed knowledge assets.

## Key Capabilities

- Secure local ingestion from `data/raw/`.
- Metadata validation with Pydantic v2.
- Deterministic section parsing and chunking.
- SQLite document/chunk registry.
- Processed JSON source of truth.
- BM25, Chroma vector, and hybrid retrieval.
- Citation builder with quote, offsets, source file, and quote hash.
- Rule-based knowledge graph extraction.
- NetworkX graph store and graph explorer.
- Query intent classification and route selection.
- Evidence-pack construction with retrieval and graph evidence.
- Optional deterministic citation-grounded answer generation.
- Structured refusal for out-of-scope, unsupported, and insufficient-evidence requests.
- Phase 6 deterministic evaluation harness and dashboard.
- Local feedback governance loop with review queue and audit events.
- Streamlit dashboard covering ingestion, exploration, graph, query, evaluation, and feedback.

## Limitations

- Synthetic enterprise corpus only.
- Deterministic local answer composer, not LLM synthesis.
- Rule-based graph extraction tuned for the demo corpus.
- Deterministic evaluation, not semantic faithfulness proof.
- Local JSONL feedback store, not a production workflow system.
- No authentication, RBAC, SSO, or production access control.
- No production monitoring or online experimentation.
- No Neo4j adapter.
- No real enterprise data integration.
- Not production-ready.

## What Is Not Implemented

- Production GraphRAG final-response synthesis.
- External LLM judge.
- Production human review workflow.
- Ticketing or SaaS workflow integration.
- Automatic feedback-to-evaluation dataset mutation.
- Retraining pipeline or automatic prompt optimization.
- Deployment infrastructure.

## Suggested Demo Narrative

1. Start with the README positioning: this is a KnowledgeOps platform, not a chatbot.
2. Show ingestion and metadata validation.
3. Show chunk traceability and citation inspection.
4. Run hybrid search.
5. Explore graph nodes and evidence quotes.
6. Use Query Planner to show intent, route, evidence pack, and optional grounded answer.
7. Run evaluation and inspect case-level results.
8. Submit feedback and triage it in the governance dashboard.
