v1.2.0-phase9b-readiness

## Summary

Phase 9B adds an Enterprise Readiness Layer to Enterprise KnowledgeOps Copilot. It keeps the platform local-first and deterministic while adding simulated enterprise personas, explainable access-policy metadata filters, a Studio Readiness page, and opt-in persona simulation for Search and Query.

This release does not add real authentication, RBAC, SSO, login sessions, production authorization, or production infrastructure.

## What Changed

- Added deterministic backend Access Policy Simulation.
- Added simulated personas for common enterprise viewing contexts.
- Added readiness summary reporting derived from local artifacts.
- Added Studio `/readiness` page for read-only readiness visibility.
- Added opt-in persona simulation controls to Search and Query.
- Preserved default Search and Query behavior when simulation is disabled.
- Kept generated filters pre-retrieval when simulation is enabled.

## APIs Added

- `GET /api/v1/readiness/personas`
  - Lists deterministic simulated personas.
- `POST /api/v1/readiness/access-policy`
  - Returns `allowed_filters`, `denied_reasons`, `explanation`, and `simulation_only: true`.
- `GET /api/v1/readiness/summary`
  - Returns readiness capabilities, corpus metadata distributions, graph/evaluation/governance status, and explicit non-goals.

## Studio Changes

- Added `/readiness` route and navigation item.
- Added readiness summary cards, persona inventory, access-policy simulation, metadata distributions, local artifact status, and boundary notes.
- Added optional Access Policy Simulation panels to:
  - Search & Citations
  - Query Planner
- Search and Query load readiness personas only when simulation is enabled.
- Search and Query apply generated metadata filters only when simulation is explicitly enabled.

## Scope Boundaries

- Simulation-only readiness layer.
- No real auth, RBAC, SSO, login, or session middleware.
- No production authorization enforcement.
- No retrieval ranking changes.
- No citation generation changes.
- No graph extraction changes.
- No evaluation behavior changes.
- No feedback governance mutation workflow added.
- No Neo4j, Redis, Kafka, Celery, Supabase/Postgres, external LLM judge, or production deployment infrastructure.

## Residual Risks

- Access Policy Simulation is a portfolio/demo signal and must not be interpreted as production authorization.
- Simulated filters depend on existing document metadata quality.
- If all simulated filters are denied, Studio blocks the Search/Query request to avoid broadening retrieval.
- The corpus remains synthetic and intentionally small.
- The readiness layer does not replace Streamlit technical dashboards.
